Native semantics

HTML elements and attributes have:

  • Roles
  • States and properties

Roles

An element's role describes its purpose

The header element

<header>...</header>
Role:
banner

The footer element

<footer>...</footer>
Role:
contentinfo

The main element

<main>...</main>
Role:
main

The nav element

<nav>...</nav>
Role:
navigation

The ul element

<ul>
<li>...</li>
<li>...</li>
</ul>
Role:
list

The li element

<ul>
<li>...</li>
<li>...</li>
</ul>
Role:
listitem

The h1 element

<h1>...</h1>
Role:
heading

The input element (checkbox)

<input type="checkbox">
Role:
checkbox

The input element (text"

<input type="text">
Role:
textbox

The select element

<select>...</select>
Role:
combobox

States and properties

An element may be in a different state, or have different properties

The checked attribute

<input type="checkbox" checked>
Role:
checkbox
State:
checked

The disabled attribute

<button disabled>...</button>
Role:
button
State:
disabled

The hidden attribute

<nav hidden>...</nav>
Role:
navigation
State:
hidden

Native keyboard support

  • Keyboard focus
  • Keyboard interaction

Interactive elements

HTML interactive elements are keyboard supported by default
https://www.w3.org/TR/html52/interactive-elements.html#interactive-elements

The a element

<a href="...">...</a>
Role:
link
State:
focusable
Interaction:
Enter

The button element

<button>...</button>
Role:
button
State:
focusable
Interaction:
Enter, Space

The select element

<select>...</select>
Role:
combobox
State:
focusable
Interaction:
Enter, up/down

Tab order

The tab order follows the DOM order of interactive elements

DOM order

<div>
    <button>1</button>
    <button>2</button>
    <button>3</button>
</div>

DOM order demo

Flexbox order

<div style="display: flex;">
    <button style="order: 3;">1</button>
    <button style="order: 2;">2</button>
    <button style="order: 1;">3</button>
</div>
